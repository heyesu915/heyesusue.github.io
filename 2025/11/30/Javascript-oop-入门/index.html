JavaScript 面向对象（OOP）新手入门教程

JavaScript 的面向对象（OOP）和 Java/C++ 等传统面向对象语言不同——JS 是基于原型（Prototype）的面向对象，而非基于类（Class）（ES6 的 class 是语法糖，底层仍基于原型）。本文从新手视角，一步步讲解 JS OOP 的核心概念和基础用法。

一、先理解：什么是“对象”？

对象是 OOP 的核心，本质是键值对的集合，用来描述一个具体的“事物”（比如一个人、一辆车）。

  • 键（key）：属性名/方法名（字符串/符号类型）
  • 值（value）：属性值（数字、字符串、布尔等）或方法（函数）

3. new 关键字做了什么？（新手必懂）

new 执行时会自动完成 4 步：

  1. 创建一个空对象 {}；
  2. 将空对象的 __proto__ 指向构造函数的 prototype（原型，下文讲）；
  3. 将构造函数的 this 绑定到这个空对象；
  4. 如果构造函数没有返回值，自动返回这个对象。

三、原型（Prototype）：解决方法复用

1. 原型的核心概念

  • 每个函数（包括构造函数）都有一个 prototype 属性（原型对象）；
  • 原型对象上的属性/方法，会被所有实例共享；
  • 每个实例都有一个 __proto__ 属性，指向构造函数的 prototype（原型链的基础）。

2. 把方法放到原型上

3. 原型链的简单理解

当访问一个对象的属性/方法时：

  1. 先在对象自身找；
  2. 找不到就去 __proto__（原型）里找；
  3. 还找不到就去原型的原型找，直到 Object.prototype（顶层）。

六、OOP 三大特性（JS 中的体现）

面向对象的核心是 3 大特性，结合 JS 总结：

1. 封装

将对象的属性和方法“包裹”在对象/类内部，隐藏内部实现，只暴露必要的接口。

  • JS 没有严格的“私有属性”（ES6+ 可用 # 定义私有属性）：

2. 继承

子类复用父类的属性和方法，减少代码冗余（上文已讲）。

3. 多态

同一方法，不同对象调用时有不同的表现（核心是「方法重写」）。

七、新手常见误区

  1. 混淆 this 指向：this 不是固定的，指向调用方法的对象（全局调用时指向 window/undefined）；
  2. 忽略原型的作用：直接在构造函数里定义方法会浪费内存，优先把方法放到原型上；
  3. 认为 class 是新的对象模型：class 只是语法糖，底层还是原型；
  4. 子类构造函数忘记 super：ES6 子类必须先调用 super() 才能用 this。

八、总结（新手学习路径）

  1. 先掌握「字面量创建对象」，理解 this 基本用法；
  2. 学习「构造函数」解决批量创建对象的问题；
  3. 理解「原型」的作用（方法复用），简单了解原型链；
  4. 重点掌握 ES6 class 和 extends（日常开发主流用法）；
  5. 理解 OOP 三大特性（封装、继承、多态）在 JS 中的体现。
